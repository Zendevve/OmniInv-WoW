# Contributing to OmniInventory

Thank you for your interest in contributing to OmniInventory! We welcome contributions from the community to help make this the best inventory addon for WoW 3.3.5a.

## Getting Started

1.  **Fork the Repository**: Create a fork of the project on GitHub.
2.  **Clone the Repo**: Clone your fork to your local machine.
3.  **Install Dependencies**: Ensure you have a WoW 3.3.5a client for testing.

## Development Workflow

1.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
2.  **Code Standards**:
    - Follow the existing code style (Lua).
    - Use 4 spaces for indentation.
    - Ensure code is compatible with Lua 5.1 (WoW 3.3.5a).
3.  **Testing**:
    - Test your changes in-game.
    - Verify that no taint issues are introduced.
    - Check compatibility with other bag addons (disable them if necessary).

## Submitting a Pull Request

1.  **Push Changes**: Push your branch to your fork.
2.  **Open a PR**: Submit a Pull Request to the `main` branch of the upstream repository.
3.  **Description**: Provide a clear description of your changes and why they are needed.
4.  **Screenshots**: If your changes affect the UI, please include screenshots.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on the GitHub repository. Provide as much detail as possible, including:
- Steps to reproduce.
- Expected vs. actual behavior.
- Screenshots or error logs (Lua errors).
