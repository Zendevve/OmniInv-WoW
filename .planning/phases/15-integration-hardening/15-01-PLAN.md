# Plan 01: Error Boundary Hardening

## Objective
Add re-entrancy guard and pcall wrapper to `Frame:UpdateLayout()` to prevent recursive calls and silent render failures. Update all event handlers to use the guarded path. This is the highest-severity gap identified in the integration scout.

## Background
The integration scout identified these specific risks:
1. `UpdateLayout()` has no re-entrancy guard — if `EQUIPMENT_SETS_CHANGED` fires during categorization, `UpdateLayout()` recurses, potentially corrupting the pool and causing C stack overflow.
2. No pcall wraps `UpdateLayout()` — if any step in the 11-step pipeline throws, the entire render aborts and the guard flag stays set, permanently locking the UI.
3. Event handlers at `Events.lua:194` and `Events.lua:260` call `UpdateLayout()` directly without guard awareness.

## Tasks

### Task 1.1: Add re-entrancy guard and pcall to UpdateLayout

<read_first>
- UI/Frame.lua (lines 1395-1600 — UpdateLayout function body)
- UI/Frame.lua (lines 1771,2090 — existing pcall patterns in render functions)
</read_first>

<action>
In `UI/Frame.lua`, wrap the body of `Frame:UpdateLayout(changedBags)` in a re-entrancy guard with pcall:

```lua
function Frame:UpdateLayout(changedBags)
    if not mainFrame or not mainFrame:IsShown() then return end

    -- Re-entrancy guard: prevent recursive UpdateLayout calls
    if self._updatingLayout then return end
    self._updatingLayout = true

    local ok, err = pcall(function()
        -- [existing UpdateLayout body goes here — lines 1398-1621]
    end)

    self._updatingLayout = nil

    if not ok then
        print("|cFFFF0000OmniInventory|r: UpdateLayout error: " .. tostring(err))
    end
end
```

Important details:
- The `pcall` wrapper must enclose the ENTIRE function body from item fetch through render, including all conditional branches (bank, alt, guildbank).
- The guard flag `self._updatingLayout` must be cleared in the outer scope (after pcall returns) so it always gets reset even if the protected call fails.
- Print the error with the addon prefix for in-game debugging.
- Keep the existing `if not mainFrame or not mainFrame:IsShown() then return end` check as the very first line (unchanged).
</action>

<acceptance_criteria>
- `UI/Frame.lua` contains `if self._updatingLayout then return end` as the second line of `UpdateLayout()`
- `UI/Frame.lua` contains `self._updatingLayout = true` immediately after the guard check
- `UI/Frame.lua` contains `local ok, err = pcall(function()` wrapping the entire function body
- `UI/Frame.lua` contains `self._updatingLayout = nil` after the pcall block
- `UI/Frame.lua` contains error print with `"|cFFFF0000OmniInventory|r: UpdateLayout error:"` prefix
- The guard flag is cleared unconditionally (not inside the pcall, not inside a conditional)
</acceptance_criteria>
