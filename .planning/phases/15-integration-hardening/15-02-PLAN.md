# Plan 02: Cross-Feature Integration Fixes

## Objective
Fix the three confirmed cross-feature issues identified by the integration scout: gear set filter / empty compression mismatch, ContextMenu virtual item fragility, and the dead `IsEquipmentSetItem()` code. Add per-cycle performance caching for gear set lookups.

## Background
The integration scout identified:
1. **Confirmed UX bug:** When a gear set filter is active (items are removed from `items` array), `CalculateEmptySlots()` scans all real bag slots and returns the unfiltered count. Empty slots shown don't match the filtered layout.
2. **Maintenance hazard:** ContextMenu uses `currentBagID`/`currentSlotID` (resolved coordinates) for actions, but also receives the unresolved `itemInfo` object which has stale source slot coordinates. If any future code reads `itemInfo.bagID` directly, it gets wrong data.
3. **Dead code:** `IsEquipmentSetItem()` in `Categorizer.lua:106-127` is defined but never called — it was replaced by `IsItemInAnySet()` in Phase 14. It uses `GetContainerItemQuestInfo(bagID, slotID)` which would break for virtual items.
4. **Performance:** `GetAllSetMemberships()` is called per-item during categorization and may trigger `BuildBlizzardSetCache()` if cache is stale. No per-cycle cache exists.

In `UI/Frame.lua`, modify `CalculateEmptySlots()` to accept an optional `items` table and, when provided, only count empty slots that correspond to the visible layout:

```lua
function Frame:CalculateEmptySlots(items)
    local emptySlots = {}
    local totalEmpty = 0

    for bagID = 0, 4 do
        local numSlots = GetContainerNumSlots(bagID)
        if numSlots then
            for slotID = 1, numSlots do
                local texture = GetContainerItemInfo(bagID, slotID)
                if not texture then
                    table.insert(emptySlots, { bagID = bagID, slotID = slotID })
                    totalEmpty = totalEmpty + 1
                end
            end
        end
    end

    -- When a gear set filter is active, only count empty slots
    -- that appear after filtered-in items in the layout
    if items and activeGearSetFilter then
        local occupiedBags = {}
        for _, item in ipairs(items) do
            if item.bagID and item.slotID then
                occupiedBags[item.bagID] = occupiedBags[item.bagID] or {}
                occupiedBags[item.bagID][item.slotID] = true
            end
        end

        local adjustedEmpty = {}
        local adjustedTotal = 0
        for _, slot in ipairs(emptySlots) do
            -- Don't skip empty slots that are between occupied slots
            -- Only skip empty slots that would have held filtered-out items
            adjustedTotal = adjustedTotal + 1
            table.insert(adjustedEmpty, { bagID = slot.bagID, slotID = slot.slotID })
        end

        -- Actually, the simpler and more correct approach:
        -- When gear set filter is active, COMPRESS all empty slots
        -- into a single count regardless of position.
        -- The layout won't match the raw bag structure anyway,
        -- so show the total empty count only.
        return {
            slots = adjustedEmpty,
            total = adjustedTotal,
            gearSetFiltered = true,
        }
    end

    return {
        slots = emptySlots,
        total = totalEmpty,
    }
end
```

**Simpler, more correct approach:** When a gear set filter is active, `CalculateEmptySlots` returns `gearSetFiltered = true`. The empty slot renderer should then render ALL empty slots as a single compressed count, ignoring bag-layout positional mapping. This avoids the complexity of trying to map filtered item positions.

Update the `UpdateLayout()` call site at line 1581:
```lua
-- Calculate empty slots for compression
self.emptySlotsData = nil
if OmniInventoryDB and OmniInventoryDB.global and OmniInventoryDB.global.enableEmptySlotCompression ~= false then
    self.emptySlotsData = self:CalculateEmptySlots(items)
end
```

And in `RenderFlowView`, when `gearSetFiltered` is true, render the empty section as a single compressed count (don't try to position-individual empty slots):
```lua
if self.emptySlotsData and self.emptySlotsData.gearSetFiltered then
    -- Render compressed empty count (ignore individual slot positions)
    -- Use the existing compressed rendering path
end
```
</action>

### Task 2.4: Add per-cycle performance cache for gear set lookups

<read_first>
- Omni/Data.lua (lines 384-410 — GetAllSetMemberships)
- Omni/Data.lua (lines 337-367 — BuildBlizzardSetCache)
</read_first>

<action>
In `Omni/Data.lua`, add a per-cycle transient cache for `GetAllSetMemberships()` and ensure `BuildBlizzardSetCache()` is never called from the categorization hot path:

```lua
-- Per-cycle cache (cleared at start of each UpdateLayout)
Data._setMembershipCache = {}

function Data:ClearSetMembershipCache()
    self._setMembershipCache = {}
end

function Data:GetAllSetMemberships(itemID)
    if not itemID then return {} end

    -- Check per-cycle cache
    if self._setMembershipCache[itemID] then
        return self._setMembershipCache[itemID]
    end

    local allSets = {}
    local seen = {}

    -- Manual sets
    local manualSets = self:GetItemGearSets(itemID)
    for _, setName in ipairs(manualSets) do
        if not seen[setName] then
            seen[setName] = true
            table.insert(allSets, setName)
        end
    end

    -- Blizzard sets (only if cache is already built)
    if self.blizzardItemToSets then
        local blizzardSets = self:GetBlizzardSetsForItem(itemID)
        for _, setName in ipairs(blizzardSets) do
            if not seen[setName] then
                seen[setName] = true
                table.insert(allSets, setName)
            end
        end
    end

    table.sort(allSets)

    -- Cache for this cycle
    self._setMembershipCache[itemID] = allSets
    return allSets
end
```

Then in `UI/Frame.lua` `UpdateLayout()`, clear the cache at the start of the render cycle (after the re-entrancy guard):

```lua
-- Clear per-cycle caches
if Omni.Data then
    Omni.Data:ClearSetMembershipCache()
end
```

Also ensure `GetBlizzardSetsForItem()` does NOT call `BuildBlizzardSetCache()` — it should only return data if cache is already built:

```lua
function Data:GetBlizzardSetsForItem(itemID)
    if not itemID then return {} end
    if not self.blizzardItemToSets then return {} end  -- Return empty, don't build
    return self.blizzardItemToSets[itemID] or {}
end
```
</action>
