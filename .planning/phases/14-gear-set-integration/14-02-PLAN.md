# Plan 02: Unified "Gear Sets" Category

## Objective
Replace the existing per-set category behavior with a unified "Gear Sets" category that contains all items belonging to any set (manual or Blizzard). Items appear once in this category, tagged with their primary set name.

## Background
Currently in `Categorizer.lua`:
- Lines 333-340: Manual sets create individual `"Set: [Name]"` categories
- Lines 348-349: Equippable items fall through to `"Equipment"`
- Blizzard sets are detected in `IsEquipmentSetItem()` (lines 106-127) but do NOT create categories — they just return `true` for equipment set detection, which doesn't affect categorization

The new behavior (from CONTEXT.md D-13 through D-16):
- All set items (manual + Blizzard) go into a single `"Gear Sets"` category
- Category priority: above `"Equipment"`, below `"New Items"` and `"Quest"`
- Item appears once, tagged with primary (first alphabetical) set name
- Tooltip still shows all memberships

### Task 2.2: Update categorization logic for unified sets

<read_first>
- OmniInventory/Omni/Categorizer.lua (lines 320-360 — existing manual set and equipment categorization)
- OmniInventory/Omni/Data.lua (the `GetAllSetMemberships()` and `IsItemInAnySet()` functions from Plan 01)
</read_first>

<action>
In `OmniInventory/Omni/Categorizer.lua`, in the `GetCategory()` function, replace the existing set category logic (around lines 333-340 and 347-349):

**Old logic to replace:**
```lua
-- Check gear sets
if Omni.Data and itemInfo.itemID then
    local sets = Omni.Data:GetItemGearSets(itemInfo.itemID)
    if #sets > 0 then
        return "Set: " .. sets[1]
    end
end

-- Equipment sets
if itemInfo.equipSlot and itemInfo.equipSlot ~= "" then
    return "Equipment"
end
```

**New logic:**
```lua
-- Check if item belongs to any set (manual or Blizzard)
if Omni.Data and itemInfo.itemID and Omni.Data:IsItemInAnySet(itemInfo.itemID) then
    return "Gear Sets"
end

-- Equipment (not in any set)
if itemInfo.equipSlot and itemInfo.equipSlot ~= "" then
    return "Equipment"
end
```

This ensures:
- Items in manual sets go to "Gear Sets" instead of "Set: [Name]"
- Items in Blizzard sets go to "Gear Sets" instead of "Equipment"
- Items that are equippable but NOT in any set still go to "Equipment"
</action>

<acceptance_criteria>
- `OmniInventory/Omni/Categorizer.lua` contains the new "Gear Sets" check before the "Equipment" check
- The old `GetItemGearSets()` → `"Set: " .. sets[1]` logic is removed or commented out
- `IsItemInAnySet()` is called to determine set membership (not just `GetItemGearSets()`)
- The `"Equipment"` fallback still exists for equippable items not in any set
</acceptance_criteria>

### Task 2.3: Add set name label to items in Gear Sets category

<read_first>
- OmniInventory/UI/ItemButton.lua (SetItem function, how itemInfo is structured)
- OmniInventory/UI/Frame.lua (how categories are rendered, especially Flow view)
</read_first>

<action>
When rendering items in the "Gear Sets" category, each item needs a visual indicator of which set it belongs to. Since items in this category may belong to different sets, the user needs to distinguish them.

Add a `primarySet` field to item info for items in the "Gear Sets" category:

In `OmniInventory/Omni/Categorizer.lua` or `OmniInventory/Omni/Data.lua`:

```lua
function Data:GetPrimarySetName(itemID)
    local allSets = self:GetAllSetMemberships(itemID)
    if #allSets > 0 then
        return allSets[1] -- First alphabetically (already sorted)
    end
    return nil
end
```

In the item info table (wherever items are enriched before rendering — likely in `GetAllBagItems()` or the categorizer), add:

```lua
if category == "Gear Sets" then
    itemInfo.primarySet = Omni.Data:GetPrimarySetName(itemInfo.itemID)
end
```

Then in `OmniInventory/UI/ItemButton.lua`, when rendering an item with `primarySet`:

Option A (minimal): Show the set name as a small text label below or above the item count.
Option B (icon/badge): Add a small colored corner indicator or icon.

**Decision:** Use Option A — show the set name as small text near the item count. This is more informative and matches the minimal-UI preference.

In `UI/ItemButton.lua`, in the `SetItem()` or render function:
```lua
if itemInfo.primarySet then
    -- Show set name as small text overlay
    -- Use a font string attached to the button, positioned below the item name/count
    button.setLabel:SetText(itemInfo.primarySet)
    button.setLabel:Show()
else
    button.setLabel:Hide()
end
```

The set label should:
- Use a small font (e.g., `GameFontNormalSmall` or custom 9pt)
- Be positioned below the item icon or in a corner
- Use the category color (light purple) or white with reduced opacity
- Truncate long names (e.g., "ProtWarrior" is fine, but "My Awesome Tank Set" → "My Awesome...")
</action>
