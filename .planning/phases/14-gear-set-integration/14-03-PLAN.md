<action>
In `OmniInventory/UI/ItemButton.lua`, modify the `OnEnter` tooltip handler to inject set membership information.

Find the tooltip code (around line 867-892). After the existing tooltip setup but before `GameTooltip:Show()`, add:

```lua
-- Add gear set information
if itemInfo and itemInfo.itemID and Omni.Data then
    local sets = Omni.Data:GetAllSetMemberships(itemInfo.itemID)
    if #sets > 0 then
        GameTooltip:AddLine(" ")
        local setList = table.concat(sets, ", ")
        GameTooltip:AddLine("Part of Sets: " .. setList, 1, 0.82, 0)
    end
end
```

**Placement:** This should appear AFTER the item's binding text (Soulbound/Binds when picked up) but BEFORE stats/effects. In WoW's `GameTooltip:SetBagItem()` flow, binding info is added automatically. We need to insert our line at the right position.

**Approach:** Since `SetBagItem()` populates the tooltip with the item's full data, and we can't easily insert between specific lines, add the set info as an additional line at the end of the tooltip (after `SetBagItem()` completes), preceded by a blank line for separation.

For `SetHyperlink()` path (offline/bank items), do the same — add after the hyperlink tooltip is populated.

The color `1, 0.82, 0` is gold/yellow (standard WoW info color).
</action>

4. Add `GetAllGearSetNames()` to Data.lua:

```lua
function Data:GetAllGearSetNames()
    local names = {}
    local seen = {}
    
    -- Manual sets
    local manualSets = self:GetGearSets()
    for setName, _ in pairs(manualSets) do
        if not seen[setName] then
            seen[setName] = true
            table.insert(names, setName)
        end
    end
    
    -- Blizzard sets
    if self.blizzardSetCache then
        for setName, _ in pairs(self.blizzardSetCache) do
            if not seen[setName] then
                seen[setName] = true
                table.insert(names, setName)
            end
        end
    end
    
    table.sort(names)
    return names
end
```

Find the filter logic (around lines 1367-1395). Add gear set filtering:

```lua
-- Apply gear set filter
if activeGearSetFilter and itemInfo.itemID then
    local sets = Omni.Data:GetAllSetMemberships(itemInfo.itemID)
    local inSet = false
    for _, setName in ipairs(sets) do
        if setName == activeGearSetFilter then
            inSet = true
            break
        end
    end
    if not inSet then
        -- Skip this item (true filter — hide non-matching)
        goto continue
    end
end
```

**Important:** The gear set filter must be independent of the existing quick filter system (`activeFilter`). A user should be able to have:
- No quick filter + Gear Set filter active = shows only items in that set
- Quick filter "Gear" + Gear Set filter active = shows only equipment items in that set
- Clear gear set filter = restores normal view
