# Plan 01: Blizzard Gear Set Cache and Event Integration

## Objective
Build a runtime cache for Blizzard Equipment Manager sets and wire up event-driven refresh. This is the data foundation that all other gear set features depend on.

## Background
OmniInventory already has a manual gear set system in `OmniInventoryDB.global.gearSets` (Tank/Heal/DPS via Ctrl+Right-click). Phase 14 adds automatic detection of Blizzard Equipment Manager sets. The addon must scan `GetNumEquipmentSets()` / `GetEquipmentSetItemIDs()` on login, cache the results, and refresh when sets change.

<action>
In `OmniInventory/Omni/Events.lua`:

1. Add `EQUIPMENT_SETS_CHANGED` to the list of events registered at addon load. The event should call `Omni.Data:BuildBlizzardSetCache()` to rebuild the cache.

2. If the event bucketing system supports it, bucket `EQUIPMENT_SETS_CHANGED` with a 0.1s delay (sets rarely change rapidly, but multiple events can fire during set management).

   ```lua

function OmniC_Equipment.GetNumEquipmentSets()
    return GetNumEquipmentSets and GetNumEquipmentSets() or 0
end

function OmniC_Equipment.GetEquipmentSetInfo(index)
    if not GetEquipmentSetInfo then return nil end
    return GetEquipmentSetInfo(index)
end

    if not GetEquipmentSetItemIDs then return nil end
```
